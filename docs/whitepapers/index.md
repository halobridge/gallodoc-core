# GalloDoc White Papers

Publication-ready white papers for GalloDoc Core v1.0 through v2.0. These documents explain the architecture behind traceable documents, governed AI execution, consent and custody metadata, AI data governance, observable agents, trust-aware decisions, agent supply chain security, and the consolidated 2.0 trusted-AI document standard.

GalloDoc Core is a release-candidate open standard and toolkit. These papers describe the public open-core model and avoid enterprise internals, customer data, PHI, secrets, and production-status overclaims.

| Paper | Theme |
|---|---|
| [GalloDoc 1.0 — From Documents to Structured Intelligence](gallodoc-1.0-from-documents-to-structured-intelligence.md) | Traceable document envelope and reusable structured intelligence. |
| [GalloDoc 1.1 — Governing AI Execution](gallodoc-1.1-governing-ai-execution.md) | Capability tokens, execution requests, execution receipts, and contracts. |
| [GalloDoc 1.2 — Consent, Custody, and Attestation](gallodoc-1.2-consent-custody-attestation.md) | Consent ledger, chain of custody, human decisions, redaction, and evidence quality. |
| [GalloDoc 1.3 — AI Data Governance and Risk Control](gallodoc-1.3-ai-data-governance-risk-control.md) | Residency, training permissions, model risk, retention, and legal hold metadata. |
| [GalloDoc 1.4 — Observable AI](gallodoc-1.4-observable-ai.md) | Agent traces, tool logs, retrieval traces, evaluations, failures, regressions, and escalations. |
| [GalloDoc 1.5 — Measuring Trust in AI Decisions](gallodoc-1.5-measuring-trust-in-ai-decisions.md) | Trust scores, decision gates, policy outcomes, recommendations, and receipts. |
| [GalloDoc 2.0 — A Standard for Trusted AI Documents, Data, and Agents](gallodoc-2.0-trusted-ai-document-standard.md) | Consolidated 2.0 standard: query, vector, relationships, versioning, policy, access, human review, workflow, connector lineage, compute trace, artifact BOM. |
